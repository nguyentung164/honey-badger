# Rules

When writing or modifying code, always prioritize:

1. High performance: avoid unnecessary re-renders, use memoization/cache appropriately, lazy load when needed, and optimize large data processing and bundle size.
2. Modern best practices: use up-to-date patterns and APIs, write type-safe, maintainable, and scalable code, and avoid deprecated approaches.
3. Consistent and polished UI: follow the existing design system, keep spacing/colors/typography/borders/shadows visually consistent, and avoid mismatched styles.
4. Thoughtful UX: make flows intuitive, provide proper loading/error/empty states, give clear user feedback, and keep primary actions easy to discover.
5. Clean code quality: keep components and functions well-structured, use clear naming, avoid over-engineering, and briefly explain important technical decisions when necessary.

Before finishing, review your changes against all the rules above and refine anything that does not meet the standard.
